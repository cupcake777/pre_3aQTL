# 05_coloc

`05_coloc/` provides the release wrapper around an external EasyColoc workflow. It converts outputs from [`03_QTL/`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/03_QTL/README.md) into EasyColoc-ready inputs and launches the external pipeline.

## Primary scripts

- [`01_prepare_inputs.py`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/01_prepare_inputs.py)
  Extracts all-pair and significant-pair QTL tables and writes coloc configuration files.

- [`02_run_easycoloc.sh`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/02_run_easycoloc.sh)
  Launches the external EasyColoc repository.

- [`02_prepare_and_run.sh`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/02_prepare_and_run.sh)
  Convenience wrapper for the two-step workflow.

## Included content

- `config/`
  Derived QTL summary metadata (`QTL_summary.csv`). EasyColoc YAML configs can be regenerated by rerunning [`01_prepare_inputs.py`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/01_prepare_inputs.py).

- `input/`
  Prepared and indexed QTL input tables.

- `gwas_subset/`
  Harmonized GWAS subsets kept with the release snapshot.

- `gwas_ma/`
  GWAS summary files in coloc/SMR-style tabular format.

## EasyColoc dependency

`05_coloc/` now resolves the external EasyColoc repository in this order:

1. `EASYCOLOC_ROOT` environment variable
2. a sibling clone at `../EasyColoc`
3. legacy local fallback paths kept for backward compatibility

For public-repo usage, the recommended pattern is:

```bash
git clone https://github.com/cupcake777/EasyColoc.git ../EasyColoc
python 05_coloc/01_prepare_inputs.py
bash 05_coloc/02_run_easycoloc.sh
```

Or set:

```bash
export EASYCOLOC_ROOT=/path/to/EasyColoc
```

## Reproducibility note

This module is not self-contained. It depends on the public EasyColoc repository plus local reference assets and a working runtime environment. [`01_prepare_inputs.py`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/01_prepare_inputs.py) regenerates [`config/QTL_summary.csv`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/05_coloc/config/QTL_summary.csv) and the EasyColoc YAML configs directly from the released [`03_QTL/`](/mnt/share_group_folder/tmp/work/Pre_3aQTL/Three_stages/03_QTL/README.md) outputs, so the checked-in summary file should be treated as a derived release artifact rather than hand-maintained metadata.
